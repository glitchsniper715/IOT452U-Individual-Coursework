# User Stories

### Primary users
1. As a bank portal, I want to verify whether a Digital ID is currently active, so that I can confirm a customer’s identity at onboarding
2. As an employee portal, I want to check if a Digital ID is valid at the time of request, so that I can verify an applicant’s identity before hiring
3. As a tax authority portal, I want to check whether a Digital ID was active throughout a reporting period so that I can validate tax submissions correctly
4. As a driving licence authority portal, I want to verify that a Digital ID is active and has no temporary restrictions, so that only eligible users can receive a licence

### Administrator / Manager
1. As a central authority officer, I want to create a new Digital ID with required attributes, so that individuals can be registered in the system
2. As a central authority officer, I want to update permitted attributes of a Digital ID, so that identity information remains accurate and up to date

### Secondary users
1. As a system staff member, I want to view audit logs of identity operations, so that I can monitor system activity and investigate issues
2. As a system staff member, I want to check the current status of a Digital ID, so that I can support organisations with verification queries

### System / Technical perspective
1. As the system, I want to enforce organisation-level authorisation on all requests, so that only permitted roles can perform management or verification actions
2. As the system, I want to validate all operations against business rules and current IS state, so that invalid or conflicting actions are rejected consistently
